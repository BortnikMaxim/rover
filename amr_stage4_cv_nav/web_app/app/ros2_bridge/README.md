# ROS 2 bridge scaffold

В этот каталог скопирована карта склада из `kolestel_rover_description/config/warehouse_map.yaml`.

## Что уже подготовлено
- backend и PWA используют ту же карту склада, что и пакет робота;
- координаты робота в API теперь трактуются как **метры в системе склада**, а фронтенд сам переводит их в пиксели;
- это позволяет позже отправлять реальные `/odom` координаты без ручной перекалибровки карты.

## Чего пока нет
В архиве робота есть описание модели, симуляция и bridge `/cmd_vel` / `/odom`, но нет готового автономного навигационного узла,
который бы принимал цели склада и сам вел робота по маршруту. Поэтому в веб-приложении оставлен симулятор маршрута,
а для реального подключения подготовлен только мост статуса.

## Следующий практический шаг
1. Поднять ROS 2 / Gazebo или реальный мини-ПК.
2. Запустить `robot_status_bridge.py`, указав адрес backend.
3. Дальше заменить симулятор на узел, который будет принимать целевую станцию и публиковать команды движения.
