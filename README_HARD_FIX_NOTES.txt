Stage 4 hard-fix build

What was simplified on purpose:
- Nav2 controller switched to DWB for a more forgiving tank / skid-steer setup.
- velocity_smoother removed from the default path to /cmd_vel.
- collision_monitor removed from default bringup.
- task_nav2_bridge now sends only the final station goal by default.
- auxiliary Gazebo helper nodes are disabled by default.
- camera / depth load reduced to keep sim responsive.

Default startup:
1) ./build_ros2.sh
2) ./run_gazebo_nav2.sh
3) ./web_app/app/backend/run_external_backend.sh
4) ./web_app/app/ros2_bridge/run_robot_status_bridge.sh
5) ./web_app/app/ros2_bridge/run_task_nav2_bridge.sh

Quick physical check before blaming Nav2:
- start only Gazebo: ./run_gazebo_only.sh
- in another terminal run: ./run_cmd_vel_smoke_test.sh

If you want the old graph-waypoint behavior:
export AMR_NAV2_USE_GRAPH_WAYPOINTS=true

If you want the alternate Nav2 parameter file:
export AMR_NAV2_PARAMS_PATH=/full/path/to/nav2_params_lidar_plus_static.yaml


Type-fix patch:
- nav2.launch.py no longer injects extra inline parameter dictionaries into controller / planner / behavior / bt / lifecycle nodes.
- controller_server should now start with a single explicit params file instead of an extra autogenerated /tmp/launch_params_*.yaml override.
